# AI 测试平台 UI/UX 优化方案

## 一、现状诊断

### 1.1 整体问题汇总

| 维度 | 当前问题 | 影响 |
|------|----------|------|
| **视觉层次** | 页面元素层次不够分明，重要信息不突出 | 用户难以快速获取关键信息 |
| **信息密度** | 大量空白区域，内容稀疏 | 页面利用率低，需要频繁滚动 |
| **操作效率** | 常用操作入口分散，需多次点击 | 影响工作流效率 |
| **数据可视化** | 仅有数字展示，缺乏趋势图表 | 无法直观感知变化趋势 |
| **引导性** | 新用户引导不足，空状态设计单一 | 上手成本高，转化率低 |
| **一致性** | 不同页面布局风格有差异 | 认知负担增加 |

### 1.2 各页面具体问题

#### 工作台 (/)
- 统计卡片设计单调，缺少视觉层次
- "最近任务"区域空状态占用过多空间
- 快捷入口按钮层次不够清晰
- 没有展示关键业务趋势

#### 提示词管理 (/prompts)
- 表格列布局需要优化，操作列过窄
- 搜索框与按钮的视觉权重相同
- 没有批量操作功能
- 标签展示不够醒目

#### 数据集管理 (/datasets)
- 存储方式列信息展示不直观
- 下载模板和上传按钮视觉权重相同
- 缺少数据预览快照

#### 模型配置 (/models)
- 卡片布局较好，但提供商分组不够清晰
- 模型状态标识太小
- 连接测试入口不够明显

#### 评估器 (/evaluators)
- Tab切换位置合理，但预置评估器卡片可点击性不明确
- 自定义评估器列表为空时没有引导
- 评估器类型标签太小

#### 测试任务 (/tasks)
- 筛选器较多但布局紧凑
- 创建任务下拉菜单发现性差
- 空状态引导需强化

#### 定时任务 (/scheduled)
- 表头信息过多
- Cron表达式可读性差
- 状态展示不够直观

#### 监控中心 (/monitor)
- 时间范围选择器位置合理
- 主区域为空时布局不协调
- 活跃告警卡片与主区域关系不清

#### 设置 (/settings)
- 左侧菜单层次结构清晰
- 表单布局合理
- 缺少保存成功的视觉反馈

## 二、优化策略

### 2.1 设计原则

1. **聚焦核心任务** - 突出"创建任务→执行→查看结果"的核心路径
2. **渐进式披露** - 基础信息优先展示，高级选项按需展开
3. **即时反馈** - 所有操作都有明确的状态反馈
4. **数据驱动** - 用可视化替代纯数字展示
5. **效率优先** - 减少点击次数，支持键盘快捷键

### 2.2 视觉规范升级

```
颜色体系升级：
├── Primary: #1677FF (保持Ant Design蓝)
├── Success: #52C41A
├── Warning: #FAAD14
├── Error: #FF4D4F
├── Info: #1890FF
│
├── 背景层次:
│   ├── Level 0: #FFFFFF (卡片)
│   ├── Level 1: #FAFAFA (页面背景)
│   └── Level 2: #F5F5F5 (分组背景)
│
└── 强调色 (新增):
    ├── Gradient: linear-gradient(135deg, #1677FF, #69B1FF)
    └── Highlight: #E6F4FF
```

#### 3.1.2 组件优化细节

**统计卡片升级**:
```tsx
// 当前: 单一数字展示
// 优化: 增加对比数据和趋势指示

<StatCard 
  icon={<FileTextOutlined />}
  title="提示词数量"
  value={12}
  trend={{ value: 2, type: 'up', period: '本周' }}
  footer="较上周 +20%"
  onClick={() => navigate('/prompts')}
/>
```

**快速开始区域**:
- 增加引导说明文字
- 主按钮使用渐变背景突出
- 次要按钮使用描边样式
- 增加快捷键提示 (如 Ctrl+N 新建任务)

**趋势图表**:
- 使用 Recharts 或 ECharts
- 支持时间范围切换 (7天/30天)
- 悬停显示详细数据
- 响应式适配
