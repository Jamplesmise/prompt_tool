## 开发原则

### MVU 原则
- 单次改动 < 5 个文件
- 单次代码 < 200 行
- 每个任务可独立验证

### 阶段依赖
```
Phase 1 ──────────────────────────────────────────────┐
    │                                                 │
    ▼                                                 │
Phase 2 ─────────────────────────────────┐           │
    │                                    │           │
    ▼                                    ▼           ▼
Phase 3                              Phase 4     Phase 5
```

### 开发流程
1. 阅读阶段的 `CONTEXT.md` 了解上下文
2. 按 `TASKS.md` 中的任务顺序开发
3. 每个任务完成后更新任务清单
4. 在开发日志中记录进度
